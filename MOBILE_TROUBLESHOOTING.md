# Решение проблем с открытием сайта на мобильном устройстве в Telegram

## Возможные причины и решения

### 1. Проблема с BotFather настройками

Убедитесь, что в BotFather правильно настроен Web App:

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Выберите вашего бота
3. Команда `/mybots` → выберите бота → `Bot Settings` → `Menu Button`
4. Убедитесь, что:
   - **Text**: название кнопки (например, "Каталог")
   - **URL**: `https://WhiteHat5.github.io/pc_salon/` (с https://)
   - URL должен быть доступен и возвращать статус 200

### 2. Проверка доступности сайта

Откройте в браузере на телефоне (не в Telegram):
```
https://WhiteHat5.github.io/pc_salon/
```

Если сайт не открывается:
- Проверьте интернет-соединение
- Убедитесь, что GitHub Pages включен в настройках репозитория
- Проверьте, что последний коммит успешно задеплоен

### 3. Проблемы с HTTPS

GitHub Pages автоматически предоставляет HTTPS. Убедитесь, что:
- URL начинается с `https://` (не `http://`)
- Сертификат валиден (зеленый замочек в браузере)

### 4. Очистка кэша Telegram

Иногда Telegram кэширует старую версию сайта:

1. Закройте Telegram полностью
2. Очистите кэш приложения (Настройки → Приложения → Telegram → Хранилище → Очистить кэш)
3. Перезапустите Telegram
4. Попробуйте открыть бота снова

### 5. Проверка консоли браузера

Для отладки на мобильном устройстве:

**Android:**
1. Подключите телефон к компьютеру через USB
2. Включите "Отладка по USB" в настройках разработчика
3. Откройте Chrome на компьютере: `chrome://inspect`
4. Выберите вкладку Telegram и откройте DevTools

**iOS:**
1. Подключите iPhone к Mac
2. Откройте Safari на Mac
3. Разработка → [Ваш iPhone] → Telegram

### 6. Проверка ошибок в коде

Откройте консоль разработчика и проверьте на наличие ошибок:
- Ошибки загрузки скриптов
- CORS ошибки
- Ошибки JavaScript

### 7. Проблемы с внешними ресурсами

Сайт использует внешние CDN:
- Tailwind CSS: `https://cdn.tailwindcss.com`
- Font Awesome: `https://cdnjs.cloudflare.com`
- Telegram WebApp SDK: `https://telegram.org/js/telegram-web-app.js`

Если эти ресурсы блокируются:
- Проверьте настройки безопасности Telegram
- Убедитесь, что интернет-соединение стабильно

### 8. Проверка viewport и мобильной версии

Убедитесь, что в `<head>` есть правильные мета-теги:
```html
<meta name="viewport" content="width=device-width,initial-scale=1,maximum-scale=1,user-scalable=no" />
```

### 9. Тестирование в разных браузерах

Попробуйте открыть URL напрямую в:
- Chrome на Android
- Safari на iOS
- Встроенный браузер Telegram

### 10. Проверка размера файла

Если `index.html` очень большой (>5MB), это может вызывать проблемы на мобильных устройствах. Проверьте размер:
```bash
ls -lh index.html
```

### 11. Проверка GitHub Pages статуса

1. Откройте репозиторий на GitHub
2. Перейдите в **Settings** → **Pages**
3. Убедитесь, что:
   - Source: `main` branch
   - Status: "Your site is live at..."

### 12. Альтернативный способ тестирования

Если проблема сохраняется, попробуйте:
1. Создать тестовый бот в BotFather
2. Установить Web App URL на тестовый домен
3. Проверить, работает ли он

## Быстрая диагностика

Выполните эти шаги по порядку:

1. ✅ Откройте сайт в браузере на телефоне (не в Telegram)
2. ✅ Проверьте, что URL правильный в BotFather
3. ✅ Очистите кэш Telegram
4. ✅ Перезапустите Telegram
5. ✅ Проверьте консоль на ошибки

## Если ничего не помогает

1. Проверьте логи GitHub Pages (если есть доступ)
2. Создайте минимальную тестовую страницу и проверьте, открывается ли она
3. Обратитесь в поддержку Telegram (если проблема специфична для Telegram WebApp)
